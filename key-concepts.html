Key Concepts

These are the main terms a beginner should understand before studying the web in more depth.

1. Web Server

A web server is either the machine or the software that stores web resources and sends them to browsers.

2. HTTP

HTTP is the main protocol used for communication between a browser and a server.

3. HTTPS

HTTPS is the secure version of HTTP. It uses encryption to protect data while it is moving across the network.

4. DNS

DNS translates readable domain names into the network addresses computers actually use.

5. Request

A request is the message sent from a browser to a server asking for a resource.

6. Response

A response is the server's reply. It includes the requested content or an error message if something fails.

7. Status Code

A status code is a numeric summary of the result of a request, such as 200 for success or 404 for not found.

Why each concept matters
Concept Why It Matters
Web Server Without a server, there is nowhere for website content to be stored and delivered from.
HTTP / HTTPS These protocols define how web data moves and how that communication is protected.
DNS DNS makes the web human friendly by letting people type names instead of numeric addresses.
Request / Response These messages make up the core cycle of web communication.
Status Code Status codes help browsers and developers understand whether a request succeeded or failed.
