Glossary and study page

Key Concepts

These terms build the foundation for understanding how web servers and HTTP work. This page expands the glossary from the earlier version of the project so the site can function more like a real study guide.

Core web communication terms

These terms appear across almost every page in the site.

1. Web Server

A web server is either the machine or the software that stores resources and returns them to clients.

2. HTTP

HTTP is the protocol used for communication between a client, such as a browser, and a server.

3. HTTPS

HTTPS is the secure version of HTTP and uses TLS encryption to protect data in transit.

4. DNS

DNS translates human-friendly domain names into IP addresses that computers can route to.

5. Request

A request is the message the client sends when it wants a resource or wants the server to perform an action.

6. Response

A response is the server's reply, including a status code, headers, and often a body with content.

Performance, security, and delivery terms

These help explain why the modern web feels fast and secure.

7. Status Code

A numeric result that tells the browser whether a request succeeded, redirected, or failed.

8. Caching

Caching stores previously fetched content so it can be reused instead of downloaded again.

9. Latency

Latency is the delay between sending a request and beginning to receive the response.

10. TLS

TLS is the encryption protocol commonly used to secure HTTPS connections.

11. Reverse Proxy

A reverse proxy sits in front of back end services and forwards requests to the right destination.

12. CDN

A content delivery network stores copies of assets in multiple locations so users can fetch them from a closer server.

Why each concept matters
Concept Why it matters
Web Server Without a server, there is nowhere for website content to be stored and delivered from.
HTTP and HTTPS These define how data moves and whether that communication is protected in transit.
DNS DNS makes the web human-friendly by letting people type names instead of raw numeric addresses.
Request and Response These two message types are the core cycle of web communication.
Status Code Status codes help both browsers and developers quickly understand what happened.
Caching and CDN These reduce repeated downloads and can move content closer to the user.
Latency and TLS These strongly affect real-world speed and security.
Reverse Proxy This helps scale, secure, and organize more complex web applications.

Study tip

If you can explain the difference between DNS, HTTP, a status code, a reverse proxy, and caching in your own words, you already understand a big part of how the web works behind the scenes.
